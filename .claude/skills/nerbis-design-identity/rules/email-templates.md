# Email Templates

## Philosophy

Transactional emails are a brand touchpoint. They should feel like they come from the same product — same voice, same restraint, same warmth. Most SaaS emails look like they were generated by a framework default. NERBIS emails should feel **intentional and premium**.

## Email Design Principles

1. **Plain over fancy.** Simple, well-structured HTML email beats a graphic-heavy template.
2. **Mobile-first.** 70%+ of emails are read on mobile. Design for 375px width.
3. **One action per email.** Every email has ONE primary CTA. Not two. Not three.
4. **Same voice as the app.** Confident, specific, concise (see micro-copy.md).
5. **Fast to scan.** User should understand the email's purpose in 3 seconds.

## Layout Structure

```
┌──────────────────────────────────┐
│  Logo (left-aligned, 120px max)  │  ← Simple, no nav bar
├──────────────────────────────────┤
│                                  │
│  Headline (what happened)        │  ← 1 line, specific
│                                  │
│  Body (context + details)        │  ← 2-4 sentences max
│                                  │
│  [ Primary CTA Button ]          │  ← One button, centered
│                                  │
│  Secondary link (text only)      │  ← Optional, understated
│                                  │
├──────────────────────────────────┤
│  Footer: company info, unsub     │  ← Minimal, legal requirements
└──────────────────────────────────┘
```

## Template Specifications

```css
/* Email body */
max-width: 560px;
background: #ffffff;  /* Always light — dark mode handled by email clients */
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
font-size: 16px;
line-height: 1.6;
color: #1c1917;  /* stone-950 — warm, readable */
```

## Email Types & Copy

### Welcome Email
```
Subject: Welcome to [Store Name]
Headline: Your store is ready
Body: [Store Name] is live at [url]. Add your first product to start selling.
CTA: [Add your first product]
```

### Order Confirmation (sent to customer)
```
Subject: Order #1234 confirmed
Headline: Order confirmed
Body: Thanks for your order from [Store Name].
      [Order summary table: product, qty, price]
      Total: $XX.XX
CTA: [View order details]
Footer note: Questions? Reply to this email.
```

### Password Reset
```
Subject: Reset your password
Headline: Reset your password
Body: Click below to set a new password. This link expires in 1 hour.
CTA: [Reset password]
Footer note: Didn't request this? You can safely ignore this email.
```

### Invite Team Member
```
Subject: [Inviter Name] invited you to [Store Name]
Headline: Join [Store Name] on NERBIS
Body: [Inviter Name] invited you to manage [Store Name]. Accept to get started.
CTA: [Accept invite]
```

### Payment Receipt
```
Subject: Payment receipt — $XX.XX
Headline: Payment received
Body: We received your payment of $XX.XX for [Plan Name].
      Next billing date: [date].
CTA: [View billing details]
```

### Subscription Cancellation
```
Subject: Subscription cancelled
Headline: Your subscription has been cancelled
Body: Your [Plan Name] subscription for [Store Name] ends on [date].
      You'll have full access until then.
CTA: [Reactivate subscription]  (secondary, not pushy)
```

## Multi-Tenant Adaptation

Emails from tenant stores should reflect the tenant's brand:

```html
<!-- Tenant logo replaces NERBIS logo -->
<img src="[tenant_logo_url]" alt="[Store Name]" style="max-height: 40px;" />

<!-- CTA button uses tenant brand color -->
<a style="background-color: [tenant_brand_color]; color: #ffffff; ...">
  View order details
</a>

<!-- Footer includes tenant info, not NERBIS -->
<p>[Store Name] · [Store Address]</p>
<p style="color: #a8a29e; font-size: 12px;">Powered by NERBIS</p>
```

**Rule:** NERBIS branding is minimal in tenant emails — small "Powered by NERBIS" in footer. The tenant's brand dominates.

## Technical Rules

1. **Inline all CSS.** Email clients strip `<style>` tags. Use `style=""` attributes.
2. **Table-based layout.** Flexbox/Grid don't work in Outlook. Use `<table>` for structure.
3. **System font stack.** Custom fonts don't load in most email clients.
4. **Images with alt text.** Many clients block images by default.
5. **CTA is a `<a>` styled as button, not a `<button>`.** Buttons don't work in email.
6. **Max-width 560px.** Readable on all screens without horizontal scroll.
7. **Preheader text.** Hidden text after subject that appears in email client preview:
   ```html
   <span style="display:none;font-size:1px;color:#fff;max-height:0;">
     Your order #1234 has been confirmed. View details →
   </span>
   ```
8. **Unsubscribe link mandatory.** Legal requirement (CAN-SPAM, GDPR).
9. **Test in Litmus or Email on Acid.** Outlook, Gmail, Apple Mail all render differently.
10. **Dark mode compatibility.** Use `color-scheme: light dark;` meta tag. Test with inverted colors.

## Copy Anti-Patterns

- "Hi [First Name]!" — no exclamation, just "Hi [Name],"
- "We're excited to..." — the user doesn't care about your emotions
- "Just a friendly reminder..." — just state the reminder
- "Don't miss out!" — pressure tactics are not premium
- "Click here" — the button text should describe the action
- Long paragraphs — 2-4 sentences per block maximum
- Multiple CTAs competing for attention
- Marketing upsells in transactional emails (legally risky, trust-damaging)
- Emojis in subject lines (unless the brand specifically calls for it)
- ALL CAPS in subject lines
